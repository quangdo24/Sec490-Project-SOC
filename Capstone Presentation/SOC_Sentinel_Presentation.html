01The Problem — Why This Project

  • SOC analysts must manually sift through thousands of Suricata IDS logs in OpenSearch just to find one alert worth investigating.
  • Each interesting log then has to be researched by hand — looking up source IPs, reputation, and geolocation across separate tools.
  • The analyst has to decide if it's a real threat — slow, judgment-heavy, and easy to get wrong under volume.
  • Finally, a MantisBT ticket is drafted manually, re-typing the same fields every time.
  • Result: repetitive, time-consuming, error-prone work with constant context-switching between OpenSearch, AbuseIPDB, and the ticketing system.

One interesting log → research → threat decision → ticket. Today that's almost entirely manual.

02Goal & Intent

  • Automate the repetitive parts of the triage pipeline — searching, enriching, and summarizing alerts.
  • Implement AI as a SOC analyst assistant — Gemini reads the raw alert and produces a plain-English incident report.
  • Automate ticket creation — the analysis is turned into a structured MantisBT ticket with one action.
  • Keep a human in the loop — the analyst verifies that the information is accurate before anything is filed.
  • Stay a learning tool, not a black box — as a student, I stay involved so I actually understand what's happening at each step.

Speed up the busywork; keep the human responsible for the decision.

04Under the Hood

// Data & Enrichment

  • Python queries OpenSearch using Lucene via the Dashboards Dev Tools proxy.
  • CIDR / wildcard IPs auto-convert to range syntax before sending.
  • Public source IPs looked up in AbuseIPDB → confidence/risk score.

// AI & Ticketing

  • gemini-2.5-flash gets a structured prompt → returns strict JSON: summary, what/why/result, recommended action.
  • Automatic PII redaction (IPs, MACs) before any data leaves for the AI — with an in-page review step in the web GUI.
  • Auto-detects the MantisBT project from the host, drafts & submits via API, and tracks submissions in My Tickets.
  • Web console extras: light/dark themes, built-in Manual tab, keyboard shortcuts, and live secret-status indicators.
  • Security built in: credentials are git-ignored, and a Bandit static scan runs in CI on every push/PR.

06Takeaways & What Got Me Here

Key Takeaways
  • Automation saves time, not judgment. SOC Sentinel handles search, enrichment, and drafting — the analyst still verifies every step.
  • Human-in-the-loop is non-negotiable. PII redaction and manual review happen before AI analysis or ticket submission.
  • One pipeline, two interfaces. CLI and web GUI share the same core logic — build once, use anywhere.
  • Security is part of the design. Git-ignored secrets, Bandit CI scans, and least-privilege API keys from day one.
  • Built to learn, not to hide. As a student project, staying involved at each stage taught me real SOC tradecraft.
