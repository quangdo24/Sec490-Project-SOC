01The Problem — Why This Project

  • SOC analysts must manually sift through thousands of Suricata IDS logs in OpenSearch just to find one alert worth investigating.
  • Each interesting log then has to be researched by hand — looking up source IPs, reputation, and geolocation across separate tools.
  • The analyst has to decide if it's a real threat — slow, judgment-heavy, and easy to get wrong under volume.
  • Finally, a MantisBT ticket is drafted manually, re-typing the same fields every time.
  • Result: repetitive, time-consuming, error-prone work with constant context-switching between OpenSearch, AbuseIPDB, and the ticketing system.
Manual workflow — no single pipeline

Goals

  • Automate the repetitive work — search, threat research, summarization, and ticket prep in one pipeline.
  • AI as a SOC analyst assistant — Gemini reads the raw alert and produces a plain-English incident report.
  • Human-in-the-loop — I verify accuracy and redact sensitive data before AI analysis or filing.
  • Built as a learning tool — stay involved at each stage to understand how the analyst workflow works.

What It Delivers

  • One tool ties together five steps that used to live in separate places.
  • Same logic powers a terminal CLI and a SIEM-style web console.
  • Color-coded Suricata severity, network flow, GeoIP, ASN, protocol, and packet/byte counts shown on every alert at a glance.
  • Each alert includes a direct link to the full Suricata log in OpenSearch Discover when you need more detail.
Human-in-the-loop checkpoint: nothing is sent to AI or filed as a ticket until I review it.

03Under the Hood

A look at how each layer of the tool is implemented — from querying the data source to filing the ticket.

// Data & Research

  • Python queries OpenSearch using Lucene via the Dashboards Dev Tools proxy.
  • Public source IPs looked up in AbuseIPDB → confidence/risk score.
  • Each alert shows which Suricata rule fired — the signature name, SID, category, and color-coded severity (1 = highest) — so you know why the log was flagged in the first place.

// AI & Ticketing

  • gemini-2.5-flash gets a structured prompt → returns strict JSON: summary, what/why/result, recommended action.
  • Automatic PII redaction (IPs, MACs) before any data leaves for the AI — with an in-page review step in the web GUI.
  • Auto-detects the MantisBT project from the host, drafts & submits via API, and tracks submissions in My Tickets.
