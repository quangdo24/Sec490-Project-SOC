    SOC Sentinel — User Manual

    Everything you need to get up and running.

    1. What this tool does

    1. Searches OpenSearch for Suricata IDS alerts via Dev Tools proxy or the cluster REST API.
    2. Enriches public source IPs through AbuseIPDB (confidence score, ISP, country, report history).
    3. Analyzes alerts with Gemini to produce a structured, plain-English incident report.
    4. Files tickets in MantisBT so the analysis lives in your case-tracking system.

    2. Before you start — credentials

    Create these JSON files inside secrets/ (examples are provided):

    • secrets/wa_opensearch.json — { "username": "...", "password": "..." }
    • secrets/abuseipdb.json — { "api_key": "..." }
    • secrets/gemini.json (optional, for AI analysis) — { "api_key": "..." }
    • secrets/mantis.json (optional, for ticket creation) — { "api_url": "...", "api_token": "..." }

    Files inside secrets/ are git-ignored; they never leave your machine.

    3. Running the tool

    • CLI: python SOC_Program.py
    • Web GUI: python SOC_Program.py --web or python web_app.py

    4. The SIEM-style alert card

    Each alert on the Alerts tab exposes the fields a SOC analyst needs at a glance:

    • Severity — Suricata severity (1 = highest) shown as a colored pill.
    • Signature / SID / category — what rule fired and why.
    • Network flow — source.ip:port → destination.ip:port with GeoIP, ASN and protocol.
    • Traffic — packets and bytes in each direction (human-readable).
    • Identifiers — community ID, flow ID, host/node, index + doc id.
    • Actions — copy IP, enrich IP via AbuseIPDB, open Discover, run AI analysis.

    5. Query tips (Lucene)

    IP field note: source.ip is not stored as a standard ip type in this index, so CIDR notation (10.0.0.0/8) and wildcards (10.*) do not work. Use bracket range syntax instead: source.ip:[10.0.0.0 TO 10.255.255.255]. The app converts IP wildcards automatically.

    Severity field: The correct field is suricata.alert.severity — not suricata.eve.alert.severity.

    QueryWhat it finds
    event.kind:"alert"All Suricata alerts
    suricata.alert.severity:[1 TO 2]High / critical alerts (sev 1–2)
    suricata.alert.severity:1Critical only (sev 1, highest)
    rule.name:ET* AND event.kind:"alert"Emerging Threats matches
    rule.name:(*C2* OR *BOTNET* OR *EXPLOIT*)C2, botnet or exploit activity
    event.kind:"alert" AND _exists_:source.ip AND NOT source.ip:[10.0.0.0 TO 10.255.255.255] AND NOT source.ip:[172.16.0.0 TO 172.31.255.255] AND NOT source.ip:[192.168.0.0 TO 192.168.255.255]External source IPs only (exclude RFC 1918)
    source.ip:"1.2.3.4"Exact source IP match
    event.kind:"alert" AND network.application:"dns"DNS-related alerts

    6. Keyboard shortcuts

    • Ctrl/⌘ + Enter — run the current query
    • / — jump to the alerts filter
    • T — toggle light/dark theme
    • 1–7 — switch tabs (Search, Alerts, IP Enrichment, AI Analysis, Tickets, My Tickets, Manual)

    7. Troubleshooting

    ProblemFix
    Red Missing secrets file toastCopy the matching .example.json in secrets/ and fill in values.
    401 UnauthorizedWrong OpenSearch username / password.
    404 from OpenSearchSet OPENSEARCH_INDEX_PATTERN to the right index, or switch transport with OPENSEARCH_TRANSPORT=cluster.
    Gemini 429You hit the free-tier rate limit — wait a minute, the app retries automatically.
